Section Three

系统架构设计 / System Architecture Design

整体架构理念
Echo 采用控制面与数据面分离的双层架构设计， 全部算力部署于用户私有边缘硬件之上，彻底解耦于第三方云服务。
Overall Architecture Philosophy
Echo adopts a Control Plane / Data Plane separation dual-layer architecture, with all compute deployed on user-owned edge hardware, completely decoupled from third-party cloud services.
📱
移动感知层
Mobile Perception Layer
低功耗音频采集与本地加密（ASR-tiny），作为数据采集前端。
Low-power audio capture with local encryption (ASR-tiny), serving as the data acquisition frontend.
🏠
局域网传输层
LAN Transport Layer
通过私有 Wi-Fi / VPN 进行安全的数据同步，数据不离开私有网络。
Secure data synchronization over private Wi-Fi / VPN, ensuring data never leaves the private network.
📦
Spring Boot 控制面
Spring Boot Control Plane
负责多设备编排、安全网络、Redis 任务队列以及 Neo4j 图谱编织。 处理多租户隔离与事务化 P2P 载荷分发。
Responsible for multi-device orchestration, secure networking, Redis task queues, and Neo4j graph weaving. Handles multi-tenant isolation and transactional P2P payload distribution.
🐍
Python FastAPI 数据面
Python FastAPI Data Plane
算法精炼厂：负责高速 WhisperX 转写、 Ollama 本地模型编排与 M4 加速 LLM 摘要处理。
The algorithmic refinery: handles high-speed WhisperX transcription, Ollama local model orchestration, and M4-accelerated LLM summarization.
📥
Docker MinIO 存储层
Docker MinIO Storage Layer
标准化本地 S3 对象存储，存放二进制原始音频与渲染后的图像明信片， 符合 S3 规范的本地仓库，承载海量数据资产与记忆快照。
Standardized local S3-compatible object store for raw binary audio and rendered image postcards. An S3-compliant local repository hosting massive data assets and memory snapshots.
🛡
解耦设计
Decoupled Design
完全独立于第三方云服务，所有模块均可在家用服务器/NAS 上独立运行。
Completely independent of third-party cloud services; all modules operate autonomously on home servers/NAS hardware.
社交图谱引擎 / Social Graph Engine — Neo4j Core
属性图模型
存储多维度的关系节点，连线本身携带内在的情感度量。 基于互动频率、情感分析与关键话题构建动态关系宇宙，映射隐性「社交引力」聚类。
Property Graph Model
Stores multi-dimensional relationship nodes, with edges carrying intrinsic sentiment metrics. Builds a dynamic relationship universe based on interaction frequency, sentiment, and key topics, mapping hidden "social gravity" clusters.
动态移动端 UI
基于 D3.js 的力导向移动端图谱视图，支持流畅的缩放、拖拽与节点扩展手势。 用户通过触摸探索自己完整的社交生态。
Dynamic Mobile UI
A D3.js-powered force-directed mobile graph view with smooth zoom, drag, and node-expansion gestures. Users explore their entire social ecosystem through touch interaction.
APOC 图算法
可直接调用本地 PageRank 与社区发现算法， 绘制隐性的社交引力聚类，揭示用户社交网络中的潜在模式。
APOC Graph Algorithms
Natively calls local PageRank and Community Detection algorithms to map hidden social gravity clusters and reveal latent patterns in the user's social network.

Section Six

联邦化 P2P 跨节点网格 / Federated P2P Cross-Node Mesh

分布式架构概述
Echo 的扩展版本支持联邦化 P2P 网格，允许多个 Echo 节点在不依赖中心服务器的情况下， 跨节点缝合各自的记忆图谱，实现真正去中心化的个人数字记忆网络。
Distributed Architecture Overview
Echo's extended version supports a federated P2P mesh, allowing multiple Echo nodes to stitch their respective memory graphs across nodes without relying on a central server, realizing a truly decentralized personal digital memory network.
🛰 零配置隧道
通过 Tailscale / WireGuard 边车容器， 跨越运营商对称 NAT，无需公网 IP 即可建立安全的跨节点连接。
🛰 Zero-Config Tunnel
Via Tailscale / WireGuard sidecar containers, traverses carrier-grade symmetric NAT to establish secure cross-node connections without requiring a public IP.
🔀 声音哈希对齐
节点凭借不可变的 SHA-256 声音签名识别共有条目， 无需统一全局 ID 即可缝合各自的图谱，实现去中心化身份认证。
🔀 Voice Hash Alignment
Nodes identify shared entries via immutable SHA-256 voice signatures, stitching their graphs together without requiring a unified global ID — enabling decentralized identity.
✉ 异步载荷分发
Spring Boot WebClient 通过事务化 Redis 重试机制， 投递经签名加密的 .echo 记忆快照，确保跨节点数据同步的可靠性与安全性。
✉ Async Payload Distribution
Spring Boot WebClient delivers signed and encrypted .echo memory snapshots via transactional Redis retry mechanisms, ensuring reliable and secure cross-node data synchronization.

RAG 驱动的记忆回想 / RAG-Powered Memory Recall

🔎 自然语言查询
用文字或语音向 APP 提问，例如：
「去年冬天我和 John 去的是哪家咖啡馆？」
「上个圣诞我们在咖啡馆聊了什么？」
即可触发实时图谱导航，获得即时摘要。
🔎 Natural Language Queries
Ask the app in text or voice, e.g.:
"Which café did John and I visit last winter?"
"What did we talk about at the café last Christmas?"
This triggers real-time graph navigation and instant summarization.
🧠 语义索引 / 向量嵌入
结合高维向量嵌入与 Neo4j 原生索引结构， 实现高级模糊语义检索，超越传统关键字匹配，理解查询的深层语义意图。
🧠 Semantic Index / Vector Embeddings
Combines high-dimensional vector embeddings with Neo4j's native index structure for advanced fuzzy semantic retrieval, going beyond keyword matching to understand the deep semantic intent of queries.
📅 每日精华（晨间胶囊）
自动生成「晨间胶囊」，凝练前一天的亮点与关系变化， 帮助用户在每日开始时快速回顾昨日的重要记忆。
📅 Daily Digest (Morning Capsule)
Automatically generates a "Morning Capsule" that distills the previous day's highlights and relationship changes, helping users quickly review key memories at the start of each day.
📐 信息蒸馏
通过算法剔除口头填充语（如"嗯"、"那个"等）， 优化上下文窗口性能与 token 熵，仅保留高价值的信息内容。
📐 Information Distillation
Algorithmically filters verbal fillers (e.g., "um", "uh", "like"), optimizing context window performance and token entropy to retain only high-value information.

AI 视觉记忆卡片 / AI Visual Memory Cards

技术原理
通过 Stable Diffusion（SDXL / LoRA） API 集成， 由对话情感峰值自动触发并生成视觉明信片。 图像原生存放于 MinIO Bucket，再以临时安全的预签名 URL 推流至用户端，节省核心带宽。
Technical Principle
Via Stable Diffusion (SDXL / LoRA) API integration, visual postcards are auto-triggered by conversational emotional peaks. Images are natively stored in MinIO Buckets and streamed to users via temporary secure pre-signed URLs, conserving core bandwidth.

Section Nine

实施路径与阶段性计划 / Implementation Roadmap

路线图总览
Echo 项目分为四个渐进式阶段，从 MVP 基础功能验证到完整生态发布，遵循开源演进路径。
Roadmap Overview
The Echo project unfolds across four progressive phases, from MVP validation to full ecosystem release, following an open-source evolution path.
1
Phase
MVP — 基础能力验证
语音活动检测（VAD）音频采集、本地加密与 Spring-MinIO 同步引擎。 实现本地语音采集与加密局域网同步引擎，完成核心数据管道的端到端验证。
MVP — Core Capability Validation
Voice Activity Detection (VAD) audio capture, local encryption, and Spring-MinIO synchronization engine. Local voice capture and encrypted LAN sync engine, completing end-to-end validation of the core data pipeline.
2
Phase
图谱与 AI 集成
Neo4j SDN 绑定及 Ollama 本地提示流水线。 核心关系映射逻辑落地，实现社交图谱的动态构建与 AI 摘要能力。
Graph & AI Integration
Neo4j SDN binding and Ollama local prompt pipeline. Core relationship mapping logic deployed, enabling dynamic social graph construction and AI summarization.
3
Phase
分布式 P2P / 视觉创作
Tailscale 跨节点隧道与声音哈希图谱缝合（扩展版）； SD 渲染与 RAG 检索系统落地（基础版）。 实现跨设备记忆同步与 AI 视觉记忆卡片生成。
Distributed P2P / Visual Creation
Tailscale cross-node tunneling with voice-hash graph stitching (extended); SD rendering and RAG retrieval system (base). Cross-device memory sync and AI visual memory card generation.
4
Phase
生态发布
ComfyUI 自定义流水线与社区插件 API 钩子。 插件 API 与社区 LoRA 模型开放，构建开放的开发者生态，赋能极客掌控自己的数字遗产。
Ecosystem Release
ComfyUI custom pipelines and community plugin API hooks. Plugin API and community LoRA model openings build an open developer ecosystem, empowering developers and geeks to own their digital legacy.
